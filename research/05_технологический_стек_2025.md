## 5.1. Введение: технологический ландшафт VR 2025

### Общая картина

Технологический ландшафт виртуальной реальности в 2025 году определяется несколькими ключевыми тенденциями, каждая из которых существенно влияет на выбор стека для культурного VR-проекта:

**1. Зрелость автономных (standalone) шлемов.** Поколение чипсетов Qualcomm Snapdragon XR2 Gen 2 и XR2+ Gen 2 обеспечивает производительность, сопоставимую с игровыми консолями начала 2020-х. Автономные шлемы полностью вытеснили PC-зависимые решения из массового сегмента. Для музейного проекта это означает принципиальную возможность развёртывания VR-экспозиций без компьютерной инфраструктуры на месте.

**2. Пассthrough и смешанная реальность (MR) как стандарт.** Цветной passthrough стал обязательной функцией всех шлемов выше $300. Это открывает для музеев технологию «дополненных экспонатов» -- наложение виртуального контента на реальные объекты без полного погружения, что снижает порог входа для неподготовленного посетителя.

**3. Конкуренция экосистем: Meta vs Pico vs Apple.** Meta Quest доминирует на потребительском рынке (~55-60% глобальных продаж standalone в 2024 году). Pico (ByteDance) активно осваивает корпоративный и азиатский сегменты. Apple Vision Pro создал премиальную нишу пространственных вычислений. Для российского рынка, находящегося под санкциями, ключевым фактором становится доступность и наличие каналов поставки.

**4. Китайские производители как альтернатива.** Pico и DPVR -- два китайских бренда, продукция которых доступна в РФ через прямые каналы и параллельный импорт без ограничений. Это делает их приоритетными кандидатами для масштабного проекта.

**5. Нейросетевая 3D-реконструкция.** Технологии NeRF (Neural Radiance Fields) и 3D Gaussian Splatting за 2023-2024 годы перешли из стадии исследований в практические инструменты. Polycam, Luma AI, Nerfstudio позволяют создавать фотореалистичные 3D-сцены из фото и видео, что радикально снижает стоимость оцифровки культурных объектов.

**6. Стандартизация: OpenXR и Universal Scene Description.** Промышленность консолидируется вокруг открытых стандартов: OpenXR как единый API для приложений, USD (Pixar Universal Scene Description) как формат обмена 3D-данными. Это снижает зависимость от конкретного вендора.

### Специфика российского контекста

Для проекта в Санкт-Петербурге технологический стек определяется дополнительными ограничениями:

- **Санкции и параллельный импорт.** Прямые поставки Meta (США) и Apple (США) в РФ прекращены. Оборудование ввозится через третьи страны (ОАЭ, Казахстан, Турция, Китай) с наценкой 15-40%. Китайские бренды (Pico, DPVR) поставляются напрямую без ограничений.
- **Валютные риски.** Курс рубля волатилен: в расчётах используется диапазон 85-100 руб./долл. Все рублёвые цены приводятся на февраль 2025 года.
- **Гарантийное обслуживание.** Для оборудования из параллельного импорта официальная гарантия не действует. Pico и DPVR имеют авторизованных дистрибьюторов в РФ (Mobi.ru, Texno.pro для Pico; DPVR Russia для DPVR).
- **Сертификация.** Для публичного пространства (музей, театр) оборудование должно иметь сертификат соответствия (декларацию ТР ТС). Pico и DPVR имеют необходимые документы, для Meta и Apple требуется дополнительная сертификация.
- **Лицензирование ПО.** Подписки на ряд SaaS-сервисов (ArborXR, ManageXR) приобретаются через международные юрлица. Российские карты работают через посредников.

## 5.2. VR-шлемы: детальная сравнительная таблица

### Методология сравнения

Для оценки шлемов используется комплекс критериев, релевантных именно для культурного проекта в музейно-театральном контексте:
- **Качество визуализации** -- разрешение, FOV, частота обновления
- **Автономность** -- время работы без подзарядки (критично для музейного сценария, где зарядка между сеансами ограничена)
- **Enterprise-функции** -- управление парком устройств (MDM/fleet management), кастомизация лаунчера, удалённое обновление контента
- **Passthrough/MR** -- качество смешанной реальности для гибридных экспозиций
- **Эргономика** -- вес, комфорт при длительном ношении, гигиеническая очистка
- **Доступность в РФ** -- каналы поставки, наличие, гарантия, цена с учётом логистики

### 5.2.3. Анализ и рекомендации по шлемам для проекта

#### Tier 1: Основные кандидаты для массового развёртывания

**Pico 4 Ultra** -- наиболее сбалансированный выбор для музейно-театрального проекта в России.
- Прямая поставка через авторизованных дистрибьюторов без санкционных ограничений
- Полноценный Pico Business Suite для fleet management: удалённое управление, развёртывание приложений, kiosk-режим, мониторинг состояния парка
- Качественный цветной passthrough для MR-экспозиций
- Процессор XR2 Gen 2 обеспечивает достаточную производительность для 360° видео 8K и интерактивных 3D-сцен
- Гарантийная поддержка в РФ
- **Рекомендация: основной шлем для проекта. Оценка: 9/10**

**Pico G3 Enterprise** -- специализированное корпоративное решение, оптимизированное для стационарных инсталляций.
- Расширенный набор enterprise-функций «из коробки»
- Встроенный NFC для идентификации пользователей
- Сменные лицевые накладки (гигиена в публичном пространстве)
- Более длительное время автономной работы (~3 ч)
- Ограничение: 3DoF-режим (без позиционного трекинга) при использовании без контроллера, хотя поддерживается и 6DoF
- **Рекомендация: для стационарных точек просмотра 360° видео. Оценка: 8/10**

**DPVR E4 (PC VR)** -- лучшее качество изображения для стационарных демонстрационных зон.
- Разрешение 2x 2560×2560 обеспечивает ~30 PPD -- один из лучших показателей в ценовой категории
- Лёгкий корпус (~280 г) -- минимальная нагрузка на шею при длительных сеансах
- Требует PC с дискретной видеокартой (NVIDIA RTX 3070 и выше)
- Прямая поставка через DPVR Russia, гарантия в РФ
- Идеален для «премиальных» точек, где качество изображения критично (например, виртуальная экскурсия по Эрмитажу в деталях)
- **Рекомендация: для стационарных премиальных точек. Оценка: 7.5/10**

#### Tier 2: Дополнительные варианты

**Meta Quest 3** -- лидер по качеству MR/passthrough и экосистеме контента.
- Лучший баланс VR/MR-возможностей на рынке
- Огромная библиотека контента и инструментов разработки
- Meta Quest for Business -- развитый fleet management
- Ограничение: только параллельный импорт в РФ, наценка 25-40%, отсутствие официальной гарантии
- Meta Quest for Business требует учётную запись Meta, что усложняет корпоративное использование в РФ
- **Рекомендация: для разработки и тестирования, не для массового развёртывания. Оценка: 7/10**

**Meta Quest 3S** -- наиболее доступный шлем с полноценными MR-возможностями.
- Цена $299 ($35-45 тыс. руб. через параллельный импорт) -- самый бюджетный вариант
- Процессор XR2 Gen 2 как у Quest 3, но с урезанным дисплеем и passthrough
- Подходит для образовательных программ с ограниченным бюджетом
- Те же ограничения параллельного импорта
- **Рекомендация: бюджетный вариант для образовательных программ. Оценка: 6.5/10**

#### Tier 3: Специализированные/нишевые

**Apple Vision Pro** -- эталонное качество, непрактичная цена.
- Лучший дисплей (micro-OLED, ~34 PPD), лучший passthrough, лучший eye tracking
- Уникальный интерфейс взаимодействия взглядом и жестами
- Категорически не подходит для массового развёртывания: цена $3 500 (400-500 тыс. руб.), хрупкость, индивидуальная калибровка, 2 часа батареи
- Может использоваться как демонстрационный/презентационный шлем (1 шт.)
- **Рекомендация: только для демонстрации руководству и VIP-показов. Оценка: 4/10 (для массового проекта)**

**HTC Vive Focus 3** -- проверенное enterprise-решение.
- Широкое поле зрения (120°), хороший дисплей (2448×2448)
- Зрелый Vive Business+ fleet management
- Высокая цена ($1 300, ~130-160 тыс. руб.), только параллельный импорт
- Устаревающий процессор (XR2 первого поколения)
- **Рекомендация: рассмотреть при наличии бюджета для премиум-точек. Оценка: 5.5/10**

**HTC Vive XR Elite** -- попытка конкурировать с Quest 3 в MR-сегменте.
- Модульная конструкция (очки + батарейный ремешок)
- Хороший цветной passthrough
- Высокая цена, ограниченная экосистема контента
- **Рекомендация: не рекомендуется для проекта. Оценка: 4.5/10**

**XREAL Air 2 Ultra** -- AR-очки, не VR-шлем.
- Лёгкие (60 г), форм-фактор обычных очков
- Подходят для лёгкого AR-наложения информации на реальную экспозицию (аудиогид с визуальными подсказками)
- Узкое поле зрения (52°), зависимость от внешнего устройства (смартфон/XREAL Beam Pro)
- Не заменяют полноценный VR-шлем
- **Рекомендация: для AR-аудиогидов и лёгких MR-дополнений. Оценка: 5/10 (для VR), 7.5/10 (для AR)**

### 5.3.2. Volumetric Capture (объёмный захват)

#### Что такое Volumetric Capture

Объёмный захват (volumetric capture) -- технология записи трёхмерного изображения объекта или человека, позволяющая зрителю в VR свободно перемещаться вокруг записанного «голографического» изображения. В отличие от 360° видео, где точка обзора фиксирована, volumetric capture обеспечивает полную свободу позиции.

#### Типы систем

**Студийные решения (многокамерные)**
- Microsoft Mixed Reality Capture Studios (закрыта в 2023): 106+ синхронизированных камер, инфракрасная подсветка
- Evercoast (ex-4DViews): от 32 камер, облачная обработка
- Volucap (Берлин): 32 камеры, 3×3 м зона захвата
- Стоимость студии: от $500 000 до $3 000 000. Стоимость аренды: от $5 000-15 000 за съёмочный день

**Портативные решения**
- Intel RealSense + софт Depthkit: бюджетный вариант, 1-4 камеры глубины
- iPhone/iPad Pro с LiDAR + Record3D: доступный захват для прототипирования
- Sony Mocopi: захват движений тела (motion capture), не геометрии
- Стоимость: от $500 (смартфон) до $20 000 (комплект Depthkit)

**NeRF/Gaussian Splatting (нейросетевая реконструкция)**
- Не требуют специальных камер: обычное видео или набор фотографий
- Polycam, Luma AI, Nerfstudio, KIRI Engine
- Качество стремительно растёт: в 2024 году Gaussian Splatting обеспечивает фотореалистичные результаты за минуты обработки
- **Наиболее перспективный подход для культурного проекта: минимальные затраты на оборудование, высокое качество результата**

**Arcturus — 4D Gaussian Splatting для объёмного захвата событий (2025-2026)**
- Стартап из Сан-Франциско (выходцы из Microsoft, ILM, Pixar, Unity) разработал платформу 4D Gaussian Splatting для захвата и трансляции живых событий в объёмном 3D-формате
- Камеры синхронизируются вокруг зоны действия, AI-алгоритмы реконструируют сцену в интерактивное 3D-представление, просматриваемое с любого ракурса в VR
- В 2025 году привлекли $2.3M seed-раунд (LDV Capital), фокус на спорте с планами расширения на концерты и театр
- **Применимость для проекта СПб:** технология прямо применима для VR-фиксации балетных постановок, театральных спектаклей и живых перформансов. Зритель в VR-шлеме сможет свободно перемещаться вокруг сцены, а не быть привязанным к одной точке обзора, как в 360° видео [источник: Telegram-каналы @cgevent, @theworldisnoteasy; LDV Capital blog, 2025]

**ActionMesh (Meta Reality Labs, 2026) — анимированные 3D-меши из видео**
- Модель от Meta для генерации анимированных 3D-мешей из моноскопического видео, текстового описания или комбинации 3D-меша с текстовым промптом
- Основана на «temporal 3D diffusion» — модификации 3D-диффузионных моделей с временной осью
- Результаты топологически консистентны и пригодны для текстурирования, ретаргетинга и импорта в любой 3D-софт
- Код и модель доступны на GitHub (facebookresearch/actionmesh) и Hugging Face
- **Применимость для проекта СПб:** быстрое создание анимированных 3D-фигур из видеозаписей танцоров, актёров и экскурсоводов для интеграции в VR-сцены без дорогостоящего volumetric capture [источник: Telegram-канал @cgevent; arXiv:2601.16148]

#### Применимость для театра и музея

| Технология | Стоимость | Качество | Применимость |
|-----------|----------|----------|-------------|
| Полноценная volumetric студия | $500K+ за студию / $5-15K за день аренды | Эталонное | Запись актёров для VR-спектаклей. Требует поездки в студию (Лондон, Берлин, Лос-Анджелес) |

## 5.4. Программное обеспечение

### 5.4.1. Игровые движки / Среды разработки VR-приложений

#### Unity 6 (Unity 2025 LTS)

- **Статус:** Основной движок для VR-разработки. Более 60% всех VR-приложений на Quest и Pico создано на Unity
- **VR-поддержка:** OpenXR, Unity XR Interaction Toolkit 3.x, ARFoundation для MR
- **Рендеринг:** Universal Render Pipeline (URP) для мобильного VR, High Definition Render Pipeline (HDRP) для PC VR
- **Ключевые VR-фичи:**
  - Foveated rendering (при наличии eye tracking)
  - Application SpaceWarp (ASW) -- интерполяция кадров для повышения плавности
  - Passthrough API для MR
  - Shader Graph для визуального программирования шейдеров
  - Unity Muse -- AI-ассистент для генерации кода и ассетов
- **Unity AI Authoring (анонс GDC, март 2026):** Unity объявила о бета-версии AI-инструмента, позволяющего создавать полноценные казуальные игры на естественном языке без написания кода. По словам CEO Мэтью Бромберга, AI-авторинг — второе стратегическое направление Unity на 2026 год. Инструмент опирается на контекст проекта и рантайм Unity, предоставляя более точные результаты, чем универсальные LLM. Для VR-культурного проекта это означает радикальное снижение порога входа в создание интерактивных VR-сцен: разработчик контента (историк, куратор) сможет создавать прототипы VR-экспозиций, описывая их на естественном языке [источник: Telegram-канал @aioftheday; Game Developer, февраль 2026]
- **Лицензия:** Unity Personal -- бесплатно (доход до $200K/год), Unity Pro -- $2 040/год за рабочее место
- **Рекомендация для проекта:** Основной движок разработки. Наибольшая экосистема плагинов, документации и специалистов на рынке труда. С анонсом AI Authoring ценность Unity как платформы для культурных проектов возрастает — команды без глубокого программного опыта смогут создавать интерактивный VR-контент

#### Unreal Engine 5.4-5.5

- **Статус:** Лидер по качеству графики. Nanite (виртуализированная геометрия) и Lumen (глобальное освещение в реальном времени) обеспечивают кинематографическое качество
- **VR-поддержка:** OpenXR, Meta XR Plugin, Pico XR Plugin
- **Преимущества:**
  - Nanite: рендеринг сцен с миллиардами полигонов без LOD-оптимизации -- идеально для фотограмметрических моделей наследия
  - Lumen: динамическое глобальное освещение -- реалистичные интерьеры дворцов
  - MetaHuman: фотореалистичные цифровые люди для виртуальных экскурсоводов
  - Movie Render Queue: высококачественный рендер для 360° видео
- **Ограничения:**
  - Более высокие требования к производительности (для standalone шлемов требуется агрессивная оптимизация)
  - Меньше VR-специалистов на российском рынке
  - Лицензия: бесплатно до $1 млн дохода, далее 5% роялти
- **Рекомендация для проекта:** Для создания премиального VR-контента (виртуальные экскурсии по дворцам с фотореалистичной графикой). Использовать в связке с PC VR (DPVR E4)

#### Godot 4.x

- **Статус:** Открытый движок (MIT-лицензия), быстро развивающийся
- **VR-поддержка:** OpenXR через плагин Godot XR Tools, базовая поддержка hand tracking
- **Преимущества:**
  - Полностью бесплатный, без роялти и ограничений
  - Лёгкий (50 МБ дистрибутив), быстрая итерация
  - GDScript -- простой язык, низкий порог входа
  - Нет зависимости от корпоративных решений (важно в контексте санкций)
- **Ограничения:**
  - VR-экосистема значительно менее зрелая, чем у Unity/UE5
  - Меньше готовых VR-плагинов и ассетов
  - Ограниченные инструменты оптимизации для мобильного VR
- **Рекомендация для проекта:** Для прототипирования и образовательных модулей. Не для основного продакшн-контента

## 5.6. Стандарты хранения и архивирования

### 5.6.1. Форматы хранения медиа

#### Изображения и текстуры

| Формат | Применение | Сжатие | Битность | Рекомендация |
|--------|-----------|--------|----------|-------------|
| **EXR (OpenEXR)** | HDR-панорамы, HDRI-карты освещения | Lossless (PIZ, ZIPS) | 16/32-bit float | Архивный формат для HDR-контента |
| **TIFF** | Высококачественные 360° фото, текстуры | Lossless (LZW, ZIP) | 8/16-bit | Архивный формат для фотоматериалов |
| **PNG** | Текстуры с альфа-каналом | Lossless | 8/16-bit | Рабочий формат для текстур UI |
| **JPEG XL** | Панорамные фото | Lossy/Lossless | 8-32 bit | Перспективный формат (постепенное внедрение) |
| **KTX2** | GPU-текстуры для VR | ASTC/ETC2/BC7 | Варьируется | Оптимальный для runtime в VR |

#### Видео кодеки

| Кодек | Статус | Аппаратное декодирование | Эффективность | Рекомендация |
|-------|--------|------------------------|--------------|-------------|
| **H.265 / HEVC** | Зрелый стандарт | Все современные шлемы | Базовая (100%) | Основной кодек для 360° видео на шлемах |
| **H.266 / VVC** | Новый (2024) | Ограничено (Samsung, Apple) | +30-50% vs H.265 | Перспективный, но пока нет аппаратной поддержки в VR-шлемах |
| **VP9** | Google | Все Android-шлемы | ~90% от H.265 | Альтернатива H.265 без лицензионных проблем |
| **AV1** | Open-source (Alliance for Open Media) | Quest 3, Pico 4 Ultra (частичное) | +20-30% vs H.265 | Перспективный, аппаратная поддержка растёт |

**Рекомендация по видео:** H.265 как основной кодек доставки (максимальная совместимость). AV1 -- для будущего контента (по мере расширения аппаратной поддержки). Архивное хранение -- ProRes 422 HQ или FFV1 (lossless).

#### 3D-форматы

| Формат | Назначение | Экосистема | Рекомендация |
|--------|-----------|-----------|-------------|
| **USD (Universal Scene Description)** | Обмен 3D-сценами | Pixar, NVIDIA Omniverse, Apple | Архивный и обменный формат для сложных сцен |
| **USDZ** | Мобильное/web отображение 3D | Apple (Quick Look), Android | Дистрибуция 3D-моделей на мобильных |
| **glTF 2.0 / GLB** | Web и runtime 3D | Khronos Group, все движки | Основной формат доставки 3D в VR-приложениях |
| **FBX** | Обмен между DCC-программами | Autodesk, Unity, UE5 | Рабочий формат обмена (проприетарный) |
| **PLY** | Облака точек, Gaussian Splats | Открытый | Формат хранения Gaussian Splatting сцен |
| **E57** | Данные LiDAR-сканирования | Открытый стандарт (ASTM) | Архивный формат для лазерных сканов |
| **OBJ** | Простые 3D-модели | Универсальный | Для простых моделей без анимации |

### 5.6.2. Стандарты и спецификации

**OpenXR (Khronos Group)**
- Единый API для VR/AR-приложений, абстрагирующий конкретное оборудование
- Поддержка: Meta Quest, Pico, HTC Vive, Windows Mixed Reality, Apple (частично)
- Для проекта: все VR-приложения разрабатывать под OpenXR для максимальной переносимости между шлемами
- Версия: OpenXR 1.1 (текущий стабильный релиз)

**MPEG-I (MPEG Immersive)**
- Набор стандартов ISO/IEC 23090 для иммерсивного медиа:
  - Part 2: Omnidirectional Media Format (OMAF) -- стандарт 360° видео
  - Part 5: Point Cloud Compression (V-PCC, G-PCC) -- сжатие облаков точек
  - Part 12: MPEG Immersive Video (MIV) -- многоракурсное видео с синтезом видов
- Для проекта: OMAF для совместимости 360° контента, V-PCC для оптимизации LiDAR-данных

**ISO 14721 (OAIS -- Open Archival Information System)**
- Эталонная модель архивирования цифровых данных
- Определяет информационные пакеты: SIP (поступление), AIP (архивное хранение), DIP (доставка)
- Для проекта: основа стратегии долгосрочного хранения VR-контента (особенно важно для культурного наследия -- контент должен быть доступен через 10-50 лет)
- Рекомендация: каждый VR-объект хранить в виде OAIS-совместимого архивного пакета: исходные файлы + метаданные + инструкция по воспроизведению

**IIIF (International Image Interoperability Framework)**
- Стандарт обмена изображениями для культурных институций (Лувр, Британский музей, Смитсоновский)
- Версия 3.0 поддерживает: изображения, аудио, видео, 3D (экспериментально)
- Для проекта: интеграция 360° панорам и 3D-моделей в общую инфраструктуру обмена данными между музеями

### 5.6.3. Стратегия архивирования для проекта

**Трёхуровневая модель хранения:**

| Уровень | Назначение | Хранилище | Формат | Срок |
|---------|-----------|----------|--------|------|
| **Горячее (Hot)** | Текущий контент на шлемах | NAS SSD, шлемы | H.265, glTF, KTX2 | Актуальная версия |
| **Тёплое (Warm)** | Рабочий архив, все версии | Облако (Yandex Cloud S3) | H.265, USD, EXR, PLY | 1-5 лет |
| **Холодное (Cold)** | Долгосрочный архив, мастер-файлы | Ленточные накопители (LTO-9) + облако (Glacier) | ProRes/FFV1, TIFF, USD, E57 | 10-50+ лет |

**Метаданные:** Dublin Core + расширение для VR (тип VR-контента, целевой шлем, версия OpenXR, пространственные координаты объекта).

### Ключевые выводы

**1. Pico как платформа первого выбора.**
Для культурного VR-проекта в России экосистема Pico (ByteDance) является оптимальным выбором по совокупности факторов: прямая доступность без санкционных ограничений, конкурентное качество оборудования (Pico 4 Ultra сопоставим с Meta Quest 3 по большинству параметров), бесплатный fleet management (Pico Business Suite), гарантийная поддержка в РФ, и наличие профессионального B2B-канала. Рекомендация: 70-80% парка шлемов -- Pico.

**2. Мультиплатформенная стратегия разработки.**
Несмотря на выбор Pico как основной платформы, контент следует разрабатывать на OpenXR через Unity 6 для обеспечения переносимости на Meta Quest, HTC Vive и другие шлемы. Это защищает инвестиции в контент от привязки к одному вендору.

**3. Нейросетевая 3D-реконструкция как технологический рычаг.**
Технологии NeRF/Gaussian Splatting радикально (в 10-100 раз) снижают стоимость создания фотореалистичных 3D-сцен по сравнению с традиционной фотограмметрией и ручным моделированием. Для культурного проекта с сотнями объектов оцифровки это критически важный фактор экономики. Годовой мониторинг Telegram-каналов (март 2025 — март 2026, 113 публикаций по GS/NeRF) подтверждает, что технология перешла в фазу рутинного производственного использования: стабильно 7-14 упоминаний/мес без резких всплесков, характерных для хайпа. Новые инновации (Triangle Splatting — 2400 FPS, Depth Anything 3, FlashWorld — 5 секунд на сцену) ещё более снижают порог входа.

**4. Гибридная модель контента.**
Оптимальная стратегия контента сочетает: (а) 360° стерео-видео для «впечатлений» (спектакли, экскурсии с живым гидом), (б) интерактивные 3D-сцены (Unity/UE5) для «исследования» (самостоятельный обход дворца), (в) MR-наложения через passthrough для «дополнения» реальной экспозиции. Каждый тип требует разного пайплайна производства.

**5. Инфраструктура определяет масштабируемость.**
Трёхуровневая архитектура (локальный NAS + облачное хранилище + CDN) обеспечивает масштабирование от одной точки до десятков без переархитектуры. Критично: качественная Wi-Fi 6/6E инфраструктура в каждой точке, так как VR-контент требует стабильной пропускной способности 100-200 Мбит/с.

**6. Бюджет реалистичен.**
Полная конфигурация проекта (3 музейные точки + производственная студия) укладывается в ~11 млн руб. (~$121 000) за оборудование и ПО. Это сопоставимо со стоимостью одной традиционной музейной экспозиции среднего масштаба, при значительно большем охвате аудитории.

### Стратегические рекомендации

**Краткосрочные (0-6 месяцев):**
- Закупить пилотный комплект: 10 × Pico 4 Ultra + NAS + Wi-Fi
- Развернуть Pico Business Suite + ArborXR
- Начать съёмку 360° контента (Insta360 Pro 2) для первых 5 объектов
- Провести R&D по Gaussian Splatting (Nerfstudio + iPhone LiDAR)

**Среднесрочные (6-18 месяцев):**
- Масштабировать до 3 точек
- Запустить производство интерактивного VR-контента (Unity 6)
- Внедрить MR-экспозиции с passthrough
- Оценить DPVR E4 для премиальных PC VR точек

**Долгосрочные (18-36 месяцев):**
- Рассмотреть следующее поколение шлемов (Pico 5, Quest 4) и Apple Smart Glasses (ожидается 2027) как AR-платформу для музейных аудиогидов
- Внедрить volumetric capture для записи театральных постановок — оценить технологию Arcturus (4D Gaussian Splatting) и ActionMesh (Meta) как альтернативы традиционным студиям
- Создать облачную платформу обмена VR-контентом между музеями
- Разработать OAIS-совместимый архив для долгосрочного хранения
- Использовать Unity AI Authoring для делегирования создания простых VR-сцен кураторам и историкам без технического бэкграунда
